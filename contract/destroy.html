Tenant Contract Destroy

OPTiM Store API Docs

Tenant Contract Setup により自動締結されたテナント契約が契約解除される場合、その旨が自動で御社サービス側の Tenant Contract API Endpoint に通知されます。

OPTiM Store -> 御社サービスへのテナント契約解除リクエスト

テナント契約解除時に、OPTiM Store から御社サービスの Tenant Contract API Endpoint に以下のような JSON データを送ります。(実際には以下の JSON に JWS 仕様にしたがって署名したデータが送られます)

{
  "iss": "https://optim-store.optim.co.jp",
  "aud": "https://your-service.example.com/contract/api",
  "contract_jwk": {
    "kid": "0-B5HVIdHUu8cPMKSn3jwlA0ctof_ajNUoblnEjYWrI"
  },
  "iat": 1470112510,
  "exp": 1470112810
}

リクエスト JSON の各要素について

key description
iss リクエスト発行者 (issuer) の識別子です。OPTiM Store の URL となります。
aud リクエスト受信者 (audience) の識別子です。御社サービスの Tenant Contract API Endpoint URL となります。
contract_jwk Tenant Contract Setup リクエスト時に OPTiM Store から送信された当該テナント契約に紐づく contract_jwk の kid が含まれます。kid 以外の鍵パラメータ (e, n etc.) は含まれません。
iat リクエスト生成日時 (issued_at) を示す Unix Timestamp です。
exp リクエストの有効期限 (expires_at) を示す Unix Timestamp です。

署名付きリクエスト

実際のリクエスト (HTTP Query String) は、上記の JSON を Tenant Contract Setup で OPTiM Store 側から送られた contract_jwk (Public Key) のペアとなる Private Key で署名されたものを、contract_statement という key の値とした Query Parameter になります。

署名に使用された鍵の ID (kid) は、JWS Payload の contract_jwk に含まれる kid および JWS Header の kid に指定されています。御社サービス側ではこの kid から解除対象となるテナント契約を特定してください。

JWS Header 例

{
  "typ": "JWT",
  "alg": "RS256",
  "kid": "0-B5HVIdHUu8cPMKSn3jwlA0ctof_ajNUoblnEjYWrI"
}

JWS Payload 例

{
  "iss": "https://optim-store.optim.co.jp",
  "aud": "https://your-service.example.com/contract/api",
  "contract_jwk": {
    "kid": "0-B5HVIdHUu8cPMKSn3jwlA0ctof_ajNUoblnEjYWrI"
  },
  "iat": 1470112510,
  "exp": 1470112810
}

署名付き JSON 例

eyJ0eXAiOiJKV1Q..(後略)..

Query String 例

GET /contract/api?contract_statement=eyJ0eXAiOiJKV1Q..(後略).. HTTP/1.1
HOST: your-service.example.com

OPTiM Store では、ユーザー企業様が企業情報や御社サービスとの間での契約ライセンス数を変更したタイミングで、上記 JSON データを御社サービスの Tenant Contract API Endpoint に HTTP DELETE で送信します。(テナント契約締結時と HTTP Method が異なる点に注意してください)

リクエスト署名の検証

Tenant Contract Setup 同様 contract_statement の署名を検証してください。

署名検証に用いる鍵には、テナント契約作成時に御社サービス側で契約と紐付けて保存しておいた contract_jwk を用います。

御社サービス -> OPTiM Store へのテナント契約解除レスポンス

上記リクエストを受け取り、署名およびリクエスト内容を検証した後、契約成立のレスポンスを返してください。

以下にレスポンス例を示します。

{
  "iss": "https://your-service.example.com/contract/api",
  "aud": "https://optim-store.optim.co.jp",
  "iat": 1464053706,
  "exp": 1464054306
}

レスポンス JSON の各要素について

key description
iss レスポンス発行者 (issuer) の識別子です。御社サービスの Tenant Contract API Endpoint URL となります。
aud レスポンス受信者 (audience) の識別子です。OPTiM Store の URL となります。
iat レスポンス生成日時 (issued_at) を示す Unix Timestamp です。
exp レスポンスの有効期限 (expires_at) を示す Unix Timestamp です。

レスポンスへの署名について

現状では御社サービス側の Contract Start API Endpoint を HTTPS 必須とすることで、契約成立レスポンスへの署名は不要としています。

契約解除処理についての注意事項

契約解除が行われても、課金処理等のためしばらくのあいだ契約内容の保存をすることがあるかと思いますが、その場合でも課金処理等が完了したのちには適切に契約内容の破棄をお願いします。

また契約解除された場合は、当該テナント契約にもとづいたライセンス付与済アカウントも削除してください。
