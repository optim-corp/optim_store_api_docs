Tenant Contract Update

OPTiM Store API Docs

Tenant Contract Setup により自動締結されたテナント契約は、ライセンス数の変化やユーザー企業担当者の連絡先等に変更があった場合にもその変更が自動で御社サービス側の Tenant Contract API Endpoint に通知されます。

OPTiM Store -> 御社サービスへのテナント契約更新リクエスト

テナント契約内容変更時に、OPTiM Store から御社サービスの Tenant Contract API Endpoint に以下のような JSON データを送ります。(実際には以下の JSON に JWS 仕様にしたがって署名したデータが送られます)

{
  "iss": "https://optim-store.optim.co.jp",
  "aud": "https://your-service.example.com/contract/api",
  "company": {
    "id": "f46f9970c914af9a22491402aa9eedc4a9645f2d2a303a39867c304ba461c47f",
    "name": "株式会社 XXX",
    "website": "https://xxx.example.com",
    "contact": "new-sysadmin@xxx.example.com"
  },
  "licence": {
    "pool": 15
  },
  "contract_jwk": {
    "kid": "0-B5HVIdHUu8cPMKSn3jwlA0ctof_ajNUoblnEjYWrI"
  },
  "iat": 1470112510,
  "exp": 1470112810
}

リクエスト JSON の各要素について

key description
iss リクエスト発行者 (issuer) の識別子です。OPTiM Store の URL となります。
aud リクエスト受信者 (audience) の識別子です。御社サービスの Tenant Contract API Endpoint URL となります。
company 実際に御社サービスを利用される OPTiM Store ユーザー企業様の情報です。id は普遍ですが、name、website、contact は更新されている可能性があるので既存の値と比較して、変更があれば適宜変更を反映させてください。
licence 購入ライセンス数等の情報です。現時点ではライセンス数のみを含みます。ライセンス数に変更があれば適宜変更を反映させてください。
contract_jwk Tenant Contract Setup リクエスト時に OPTiM Store から送信された当該テナント契約に紐づく contract_jwk の kid が含まれます。kid 以外の鍵パラメータ (e, n etc.) は含まれません。
iat リクエスト生成日時 (issued_at) を示す Unix Timestamp です。
exp リクエストの有効期限 (expires_at) を示す Unix Timestamp です。

署名付きリクエスト

実際のリクエスト (HTTP Request Body) は、上記の JSON を Tenant Contract Setup で OPTiM Store 側から送られた contract_jwk (Public Key) のペアとなる Private Key で署名されたものを、テナント契約作成時同様 contract_statement という key に wrap した JSON データになります。

署名に使用された鍵の ID (kid) は、JWS Payload の contract_jwk に含まれる kid および JWS Header の kid に指定されています。御社サービス側ではこの kid から更新対象となるテナント契約を特定してください。

JWS Header 例

{
  "typ": "JWT",
  "alg": "RS256",
  "kid": "0-B5HVIdHUu8cPMKSn3jwlA0ctof_ajNUoblnEjYWrI"
}

JWS Payload 例

{
  "iss": "https://optim-store.optim.co.jp",
  "aud": "https://your-service.example.com/contract/api",
  "company": {
    "id": "f46f9970c914af9a22491402aa9eedc4a9645f2d2a303a39867c304ba461c47f",
    "name": "株式会社 XXX",
    "website": "https://xxx.example.com",
    "contact": "new-sysadmin@xxx.example.com"
  },
  "licence": {
    "pool": 15
  },
  "contract_jwk": {
    "kid": "0-B5HVIdHUu8cPMKSn3jwlA0ctof_ajNUoblnEjYWrI"
  },
  "iat": 1470112510,
  "exp": 1470112810
}

署名付き JSON 例

eyJ0eXAiOiJKV1Q..(後略)..

Request Body 例

{
  "contract_statement": "eyJ0eXAiOiJKV1Q..(後略).."
}

OPTiM Store では、ユーザー企業様が企業情報や御社サービスとの間での契約ライセンス数を変更したタイミングで、上記 JSON データを御社サービスの Tenant Contract API Endpoint に HTTP PUT で送信します。(テナント契約締結時と HTTP Method が異なる点に注意してください)

リクエスト署名の検証

Tenant Contract Setup 同様 contract_statement の署名を検証してください。

署名検証に用いる鍵には、テナント契約作成時に御社サービス側で契約と紐付けて保存しておいた contract_jwk を用います。

御社サービス -> OPTiM Store へのテナント契約更新レスポンス

上記リクエストを受け取り、署名およびリクエスト内容を検証した後、契約成立のレスポンスを返してください。

以下にレスポンス例を示します。

{
  "iss": "https://your-service.example.com/contract/api",
  "aud": "https://optim-store.optim.co.jp",
  "iat": 1464053706,
  "exp": 1464054306
}

レスポンス JSON の各要素について

key description
iss レスポンス発行者 (issuer) の識別子です。御社サービスの Tenant Contract API Endpoint URL となります。
aud レスポンス受信者 (audience) の識別子です。OPTiM Store の URL となります。
iat レスポンス生成日時 (issued_at) を示す Unix Timestamp です。
exp レスポンスの有効期限 (expires_at) を示す Unix Timestamp です。

レスポンスへの署名について

現状では御社サービス側の Contract Start API Endpoint を HTTPS 必須とすることで、契約成立レスポンスへの署名は不要としています。

契約ライセンス数変更処理についての注意事項

OPTiM Store では、契約ライセンス数以上のライセンス付与済アカウント数が発生しなように Validation を行っていますが、御社サービス側での非同期処理やライセンス剥奪通知処理の失敗等により、ライセンス付与済アカウント数が契約ライセンス数を上回る可能性はゼロではありません。

もしライセンス付与済アカウント数が契約ライセンス数を上回るような状況が発生する場合には、ライセンス数変更通知に対するレスポンスとして 400 エラーを返すようにしてください。

400 エラーを検知した場合、OPTiM Store 運営サイドより御社サービス担当者様にテナント契約内容のズレを修正するべくご連絡させていただきます。
